Schema Validator

Paste or type a YAML/JSON document below, select the construct and version, and validate it against the published schema.

Results will appear here after validation.
How it works

The validator fetches the construct's entity schema (<construct>.yaml) from the repository, parses your document, and checks it against the schema using client-side JSON Schema validation. All processing happens in your browser — nothing is sent to a server.

What Gets Checked

  • Required fields — are all required properties present?
  • Additional properties — schemas use additionalProperties: false, so unknown fields are rejected.
  • Types — does each field match its declared type?
  • Formats — do uuid, date-time, and other format constraints pass?
  • Constraints — minLength, maxLength, minimum, maximum, pattern
  • Enum values — is the value one of the allowed options?

Limitations

  • Cross-file $ref resolution is limited — the validator resolves refs within the entity schema but may not follow deep external references.
  • This validates structure, not semantics. A valid document may still be rejected by the server for business-logic reasons.
  • For full server-side validation, use the Meshery API directly.
